# Documentation

This folder contains all project documentation organized by category.

## Folder Structure

```
docs/
├── api/                    # API documentation and testing guides
├── migration/             # Database migration guides and references
├── DEVOPS_INTEGRATION.md  # DevOps and CI/CD setup
├── PRODUCTION_CHECKLIST.md # Pre-deployment checklist
└── PROJECT_SUMMARY.md     # Overall project documentation
```

## Documentation Categories

### API Documentation (`api/`)
- API testing guides
- Endpoint documentation
- Request/response examples
- Postman/REST client guides

### Migration Documentation (`migration/`)
- Database migration setup
- Migration usage guide
- Schema change procedures
- Rollback documentation

### General Documentation
- **DEVOPS_INTEGRATION.md**: Docker, CI/CD, deployment workflows
- **PRODUCTION_CHECKLIST.md**: Pre-production verification steps
- **PROJECT_SUMMARY.md**: System architecture and overview

## Root Documentation

The following files remain in the backend root for quick access:

- **README.md**: Main project documentation and quick start
- **QUICKSTART.md**: Fast setup for new developers
- **.env.example**: Environment variable template

## Contributing

When adding new documentation:

1. Choose the appropriate folder based on content type
2. Use clear, descriptive filenames
3. Follow existing markdown formatting
4. Update this README if adding new categories
